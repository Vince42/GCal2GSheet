# Codex Instructions

## Development Process

The agent must prove conformance before implementation, not explain correctness afterward.

Before modifying code or behavior, the agent must:

1. Identify affected ADRs.
2. Identify affected invariants.
3. Identify acceptance tests.
4. Identify regression risks.
5. Describe the implementation plan.

Only after completing these steps may implementation begin.

For documentation-only changes, the agent may use a reduced analysis that identifies affected documents, relevant ADRs, and documentation validation checks before editing.
